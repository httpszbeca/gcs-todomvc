# Diretrizes de Contribuição

## Branches
- Palavras separadas por underline: `correcao_escape_html`
- Funcionalidades: `nome_da_funcionalidade`
- Correções: `correcao_do_bug`
- Hotfix: `correcao_rapida_xyz`

## Commits
Formato: `tipo: descrição no presente, 3ª pessoa, sem ponto`

| Tipo     | Uso                              |
|----------|----------------------------------|
| feat     | Nova funcionalidade              |
| fix      | Correção de bug                  |
| doc      | Documentação                     |
| refactor | Refatoração sem impacto lógico   |
| conf     | Arquivos de configuração         |
| update   | Atualização de dados             |
| del      | Remoção de arquivos              |
| merge    | Resolução de conflito de merge   |

## Pull Requests
- Todo PR deve ter ao menos 1 aprovação antes do merge
- Descreva o que foi feito e qual issue é resolvida
